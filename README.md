# 🕒 실시간 경매 시스템 (SSE-Auction)
Server-Sent Events(SSE) 기술을 활용하여 구현한 단방향 실시간 웹 경매 애플리케이션입니다.
웹소켓(WebSocket) 없이 순수 HTTP 통신과 SSE만을 조합하여 양방향 통신과 유사한 실시간 기능을 구현한 것이 특징입니다.
## ✨ 주요 기능
- **실시간 타이머 동기화**: 서버에서 관리하는 카운트다운 타이머가 모든 클라이언트에게 실시간으로 동기화됩니다.
- **실시간 입찰 및 최고가 갱신**: 사용자가 입찰 시, 접속해 있는 모든 사용자의 화면에 현재 최고가와 입찰 내역이 즉시 반영됩니다.
- **경매 종료 알림**: 타이머가 0이 되면 경매가 종료되며, 오버레이 팝업을 통해 낙찰자와 최종 가격을 안내합니다.
## 🛠 기술 스택
- **Backend**: Node.js, Express.js
- **Frontend**: Vanilla HTML/CSS/JavaScript (Glassmorphism UI 적용)
- **Real-time Communication**: 
  - `Server-Sent Events (EventSource)`: 서버 ➡️ 클라이언트 단방향 실시간 푸시
  - `Fetch API (POST)`: 클라이언트 ➡️ 서버 데이터 전송
## 💡 개발 포인트 (WebSocket vs SSE)
이 프로젝트는 양방향 통신이 필요한 기능(경매)을 **SSE(단방향 푸시) + HTTP POST 요청** 구조로 분리하여 설계했습니다.
1. **서버 응답 (SSE)**: 서버에서 타이머, 입찰 현황, 경매 종료 등의 상태 변화를 `res.write()`를 통해 클라이언트로 밀어넣어 줍니다.
2. **클라이언트 요청 (HTTP POST)**: 클라이언트에서 '입찰하기' 버튼을 누르면 일반적인 Fetch API를 통해 서버로 전송합니다.
3. 이를 통해 웹소켓 의존성(`ws` 패키지) 없이 내장 API만으로도 훌륭한 실시간 동기화 UI를 구현할 수 있음을 확인했습니다.
